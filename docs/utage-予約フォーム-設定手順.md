# UTAGE 予約フォーム 設定手順書（SiiD 個別相談）

Jicoo から UTAGE への切り替えにあたり、**UTAGE 管理画面（GUI）で実施いただく作業**の手順書です。
予約フォーム本体・リマインド配信・LINE連携・Zoom連携は UTAGE の MCP（API）では設定できないため、管理画面での操作が必要です。

> 目的: Jicoo 経由のドタキャン率を、**申込後の LINE リマインド自動配信**で低減すること。

---

## 前提

- UTAGE アカウント（保有済み）
- 営業に使う Zoom アカウント（保有済み）
- LINE 公式アカウント（リマインドを LINE で送る場合に必要。未保有ならメールのみでも可）

---

## 全体の流れ

```
1. 予約フォーム（イベント/個別相談）を作成
2. Zoom を連携（予約確定で Zoom URL 自動発行）
3. リマインド配信シナリオを作成（LINE / メール）
4. 予約完了をトリガーにリマインドを紐づけ
5. 埋め込みタグ / 予約URL を取得 → Claude に共有
```

---

## Step 1. 予約フォーム（個別相談）の作成

1. UTAGE 管理画面 →「**イベント管理**」（説明会・セミナー・個別相談の予約管理）を開く。
2. 新規イベントを作成し、種別を「**個別相談**」相当に設定。
3. 基本情報を入力:
   - イベント名: `SiiD 無料個別相談`
   - 所要時間: **60〜90分**（現行LPの記載「所要60-90分」に合わせる）
   - 開催形式: **オンライン（Zoom）**
4. **予約可能な日時枠（カレンダー/タイムスロット）** を設定（営業可能な曜日・時間帯、1日の上限件数、予約締切リードタイムなど）。
5. 申込フォーム項目を設定（氏名・メール・電話 など。LINE リマインドを使う場合は **LINE 友だち登録への導線/連携項目**を必ず含める）。

---

## Step 2. Zoom 連携

1. UTAGE の外部連携設定で **Zoom アカウントを接続**。
2. 予約確定時に **Zoom ミーティング URL が自動発行**され、申込者へ通知されるよう設定。
3. 完了通知（サンクスメール/画面）に Zoom URL が入ることを確認。

> 現行の `counseling-complete` ページは「Zoom URL はメールで送付」という案内のため、UTAGE 側の自動通知と齟齬がないようにする。

---

## Step 3. リマインド配信シナリオの作成（ドタキャン対策の核）

1. UTAGE 管理画面 →「**配信**」で配信アカウントを用意。
   - LINE で送る場合: アカウント種別 **`line`** または **`mail_line`**（LINE+メール併用）
   - メールのみ: **`mail`**
2. **LINE 公式アカウントを連携**（LINE で送る場合）。
3. リマインド用の **配信シナリオ（ステップ配信）** を作成。推奨タイミング例:
   - 予約直後: 予約確定＋Zoom URL の案内
   - **前日**: リマインド（日時・Zoom URL 再掲）
   - **当日（開始1〜2時間前）**: 直前リマインド
4. 各メッセージに **日時・Zoom URL・キャンセル/変更方法** を明記。

---

## Step 4. 予約完了とリマインドの紐づけ

1. Step 1 の予約フォームの **予約完了（申込）をトリガー** に、Step 3 のリマインドシナリオが自動開始するよう設定。
2. LINE 友だち登録済みの申込者へ LINE で、未登録者へはメールでリマインドが届くよう経路を確認。

---

## Step 5. 埋め込みタグ / 予約URL の取得 → Claude へ共有

1. 作成した予約フォームの **外部サイト向け埋め込みコード**（iframe など）または **予約ページの公開URL** をコピー。
2. 以下を Claude に共有してください:
   - [ ] 埋め込みタグ（iframe / scriptタグ）**または** 予約ページURL
   - [ ] 「フォームが表示されない場合」用のフォールバックURL（予約ページの直リンク）
   - [ ] 推奨表示サイズ（あれば。なければ現行の `height:720px` を踏襲）

→ 受領後、Claude が以下2箇所の Jicoo 埋め込みを UTAGE に差し替えます:
- `src/pages/siid/counseling/index.html`
- `src/pages/siid/lp-1/index.html`（LP の `#cta` セクション）

---

## 動作確認（設定後）

1. 予約フォームでテスト予約を実施。
2. Zoom URL が自動発行・通知されることを確認。
3. **LINE（および/またはメール）でリマインドが届く**ことを確認。
4. サイト差し替え後、`/siid/counseling/` と `/siid/lp-1/` で UTAGE フォームが正しく表示されることを確認。

---

## 補足

- UTAGE の **MCP（API）でできること**: ファネル/ページ作成、配信アカウント/シナリオの作成・参照。配信シナリオの雛形作成など一部は Claude が MCP で補助可能です。ご希望あればお申し付けください。
- **MCP でできないこと**: 予約フォーム本体（時間枠）、リマインドの最終設定、LINE連携、Zoom連携、埋め込みタグ取得 → 本手順書の GUI 作業で対応。
