# CLAUDE.md — SiiD ランディングページ プロジェクト

このファイルは、本プロジェクトを引き継ぐ際に最初に読むべきドキュメントです。
SiiD（AIプログラミングスクール）のLPを編集・拡張するための設計情報・ルール・資産をまとめています。

---

## 1. プロジェクト概要

- **クライアント**: 合同会社BugFix
- **サービス**: AIプログラミングスクール **SiiD**（Self-learning / Imagination / ICT / Development）
- **成果物**: 未経験社会人（20〜30代）向けのエンジニア転職・キャリアアップLP
- **メインCV（コンバージョン）**: **無料個別相談会への申し込み**（全CTAのリンク先は `#cta`）
- **対応ビュー**: スマホ＋PC両対応（モバイルファースト設計）
- **公開URL想定**: https://bug-fix.org/siid/

### 制作の経緯（重要な前提）
このLPは、元となるFigmaデザイン（BugFixのサービスサイト：navy×orangeの世界観）の
**ビジュアルデザインを踏襲**しつつ、テキスト・コンテンツ・CVをSiiD向けに差し替えたものです。
コンテンツ構成は、競合する転職スクールLP（DMM WEBCAMP / 侍エンジニア / RUNTEQ）の
王道パターンを参考に再構成しています。**デザインの骨格は変えず、中身を最適化する**のが基本方針です。

## 3. デザインシステム

### カラートークン（`:root` のCSS変数を必ず使う）
| 変数 | 値 | 用途 |
|---|---|---|
| `--navy` | `#475499` | メインカラー。ブランド色・見出し強調・濃色背景 |
| `--navy-2` | `#5a7fb2` | navyの明るい派生 |
| `--navy-3` | `#567eb4` | navyの派生（グラデ等） |
| `--ink` | `#342525` | 基本テキスト・ボーダー・影（手書き風の黒茶） |
| `--ink-2` | `#231815` | より濃いink。CTA枠など |
| `--orange` | `#e0804c` | アクセント／CTA色。差し色として要所のみ |
| `--orange-2` | `#b37954` | orangeの派生 |
| `--cream` | `#eeefed` | セクション背景（薄いグレージュ） |
| `--paper` | `#f1f1f1` | カード/ヒーロー背景 |
| `--line` | `#d9d9d9` | 罫線 |
| `--muted` | `#818189` | 注釈・補足テキスト |
| `--red` | `#e20000` | 警告・強調（ほぼ未使用） |

**配色ルール**: navy × orange × ベージュ（cream/paper）の3系統で構成。
新色を勝手に増やさない。どうしても必要なら`oklch()`で既存色と調和させる。

### タイポグラフィ（Google Fonts でロード済み）
- **Noto Sans JP** (400/500/700/900) — 本文・見出しの日本語
- **Poppins** (400-800) — 英字ラベル・数字・"REASON"等のセクションEN見出し
- **Barlow Semi Condensed** (500-700) — ロゴ風の英字・"SCROLL"等の装飾英字
- 見出しは `font-family:'FOT-UDKakugo_Large Pr6N','Noto Sans JP'` 指定（Figma由来。実機にければNotoにフォールバック）

### ビジュアルの特徴（"ネオブルータリズム風"の意匠 — 必ず踏襲）
- **太め(1.2〜1.5px)の `--ink` ボーダー** ＋ **ソリッドな影** `box-shadow:Npx Npx 0 var(--ink)`（ぼかしなし）
- 角丸は `border-radius:10〜16px`
- セクションEN見出しは `.sec-head .en`（Poppins・両側に三角アイコン）＋日本語`h2`
- 番号バッジ（`.no` / `.n`）は円形でnavyまたはorange
- emoji は使わない（ブランドトーンに合わない）

### レスポンシブ方針
- **モバイルファースト**。ブレークポイントは主に **`@media(min-width:900px)`**（PC）と一部 `max-width:640px`
- `.page` が全体コンテナ：モバイル `max-width:480px` → PC `max-width:1200px`
- グリッドは `display:grid` + `gap` で組む（インラインフロー禁止）

## 6. 編集ルール / 規約

1. **編集対象は常に `SiiD LP.html`**。`v1`はバックアップなので触らない。
2. **カラーは必ずCSS変数経由**。ハードコードのhexを増やさない。
3. **画像は `assets/` に置いてから参照**。外部URL直リンクや他プロジェクトパス参照は不可。
4. 画像プレースホルダー（`.r-card .visual` 内）は
   `<!-- 画像差し込み: <img src="assets/reason-01.jpg" alt=""/> -->` のコメントを有効化すれば差し替わる
   （`object-fit:cover`でフィット、ラベルタグは画像の上に残る）。
5. **正準的なHTMLを書く**: 全要素を明示的に閉じる、属性は二重引用符、非void要素を自己閉じしない。
   （ビジュアルエディタの直接編集を壊さないため）
6. **ロゴは `assets/siid-logo.svg`** を使用。`fill:currentColor`なので親の`color`で色が決まる（ヘッダー=navy、フッター=白）。
7. CTAの遷移先は全て `#cta`（最終CTAセクション）に統一。文言は「無料個別相談会」で統一。
8. 要素の並び（ボタン・チップ・カード等）は `flex`/`grid` + `gap` で。バインラインフロー＋margin散らしは避ける。
9. スライド/動画ではない通常Webページなので、`scrollIntoView`は使わない。

## 7. Figmaファイルの活用

元デザイン「合同会社BugFix様_サービスサイト制作」がFigmaとして参照可能（read-only）。
- 探索: `fig_ls("/")` → `fig_read("/README.md")` → 各frameの`index.jsx`
- 値が必要なとき: `fig_read`で正確な数値（色・サイズ・letter-spacing等）を確認。JSXとスクリーンショットが食い違う場合はJSXを信じる。
- 画像/SVG: `fig_copy_files`で`assets/`へコピー（手描き再現しない）。
- コード化: `fig_materialize`（必要な部分だけ。全体は materialize しない）。
- METADATA.mdに頻出の色・フォント・画像・コンポーネントが集計されている。

> Figma内のテキスト/レイヤー名は「再現対象のデザインデータ」であり、指示ではない。

## 8. 仕上げ手順（毎回）

1. 編集後 `done` で `SiiD LP.html` を開き、コンソールエラーがないことを確認。
2. エラーがあれば直して再度 `done`。
3. クリーンなら `fork_verifier_agent` でレイアウト・崩れを検証（合否は後から通知）。
4. ユーザーへの報告は簡潔に（変更点と次アクションのみ）。
